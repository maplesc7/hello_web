# Monolith Timer Specification

## 1. Purpose
Build a single-file countdown timer UI with a sci-fi/glass panel style, supporting start/pause/resume/end flow and notification fallback behavior.

## 2. Output File
- Target: `timer/index.html`
- Single HTML with internal CSS/JS
- No external libraries

## 3. Theming
Use CSS custom properties with two modes:
- Default dark (root)
- Light override via `[data-theme="light"]`

Theme selector options:
- `dark`
- `light`
- `device` (follow `prefers-color-scheme`)

Persist selected mode in localStorage key:
- `monolith_theme`

## 4. Layout
- Centered app shell with max width around 920px
- `.app` is scrollable if needed (`overflow: auto`)
- Main stage panel contains visual effects and timer controls

## 5. Visual Design
Include layered effects:
- Gradient page background
- Glass panel with border and blur
- Inner ring borders via `.app::before/.app::after`
- Stage overlays:
  - `.wire-grid`
  - `.aurora-flare`
  - `.aurora-rays`
  - `.particles` (3 static points)

## 6. Timer Dial
- Circular dial (`.dial`) with 60 segment ticks
- Segment nodes are generated by JS (`setupSegments`)
- Activated segments indicate elapsed progress
- Center text shows `MM:SS`

## 7. Inputs
Numeric inputs:
- Minutes: 0..99
- Seconds: 0..59

Rules:
- Clamp invalid values
- Sync displayed values with timer state
- Inputs editable only when status is `stopped` or `finished`

## 8. State Model
Statuses:
- `stopped`
- `running`
- `paused`
- `finished`

State fields:
- `durationSec`
- `remainingSec`
- `intervalId`
- `endAtMs`
- `toastTimer`

## 9. Behavior
### 9.1 Start
- Parse inputs
- Reject `<= 0` with toast (`set 1 sec or more`)
- Initialize duration/remaining
- Start interval ticking every 200ms

### 9.2 Tick
- Remaining time computed from wall clock (`endAtMs - Date.now()`)
- Ceil to seconds
- On zero, finish

### 9.3 Pause
- Only from running
- Stop interval
- Keep remaining seconds

### 9.4 Resume
- Only from paused
- Recompute `endAtMs`
- Restart interval

### 9.5 End
- Return to stopped
- Restore remaining to full duration

### 9.6 Finished
- Status becomes `finished`
- Progress ring full
- Trigger notification/toast

## 10. Controls Rendering
Controls are rendered dynamically based on state:
- Stopped: `▶ start`
- Running: `⏸ pause`, `■ end`
- Paused: `▶ resume`, `■ end`
- Finished: `▶ resume` (restart from current inputs)

## 11. Notification Handling
- Button requests notification permission
- Show toast if unsupported or denied
- Requirements for system notification:
  - Secure context
  - `Notification` API support

On finish:
- Send system notification if allowed
- Always show toast (`timer finished`)

## 12. Accessibility
- `aria-live` on stage/toast regions
- Buttons include `title` and `aria-label`
- Focus-visible outlines for interactive controls

## 13. Responsive Rules
- Under 520px:
  - Narrow dial
  - Stack action row
  - Controls become single column
  - Theme select expands full width

## 14. Key Functions
Implement equivalents of:
- `setupSegments`
- `parseInputs`
- `syncInputs`
- `setProgress`
- `renderTime`
- `showToast`
- `startWith`
- `tick`
- `pauseTimer`
- `resumeTimer`
- `endToStopped`
- `renderControls`
- `applyTheme`
- `initTheme`

## 15. Non-Functional Requirements
- Keep implementation compact and deterministic
- Avoid framework dependencies
- Preserve tabular numeric readability
